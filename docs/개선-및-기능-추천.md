# 치즈스테레오 개선 및 기능 추천

앱 구조·코드 점검을 바탕으로 한 개선 제안과 기능 추천입니다. 우선순위는 팀 상황에 맞게 조정하면 됩니다.

---

## 1. 개선 제안 (우선 적용 권장)

### 1.1 조회수 표시

- **현상**: `VideoCard`에서 `video.viewCount`를 표시하지만, YouTube **Search API**는 `statistics`를 주지 않아 현재는 항상 비어 있을 수 있음.
- **제안**:  
  - **A)** 영상 탭만이라도 조회수를 쓰고 싶다면, 검색/목록 응답의 `videoId` 목록으로 **videos.list** (`part=statistics`) 한 번 더 호출해 조회수를 채우기.  
  - **B)** API 호출 수를 늘리고 싶지 않다면, 조회수 영역을 제거하거나 “조회수순 정렬 지원” 정도만 문서에 명시.

### 1.2 네트워크·오프라인 처리

- **현상**: 인터넷이 끊기면 API 실패 시 “비디오를 불러오는데 실패했습니다” 정도만 나옴.
- **제안**:  
  - `expo-network` 또는 `@react-native-community/netinfo`로 **연결 상태** 확인.  
  - 오프라인/타임아웃일 때 **“인터넷 연결을 확인해 주세요” + 재시도 버튼** 등으로 메시지 구분.

### 1.3 플레이어 화면 `id` 처리

- **현상**: `useLocalSearchParams`의 `id`가 `string | string[]`일 수 있어, URL에 따라 재생 실패 가능.
- **제안**:  
  - `id`가 배열이면 `id = Array.isArray(id) ? id[0] : id` 로 한 번 정규화한 뒤 `YoutubePlayer`에 넘기기. (이미 제목은 `resolvedTitle`로 처리된 상태라, id만 동일하게 처리하면 됨.)

### 1.4 음악/영상 탭 코드 중복

- **현상**: `music.tsx`와 `videos.tsx`가 목록 로드·검색·정렬·재생 선택 로직이 거의 동일함.
- **제안**:  
  - **공통 훅** `useChannelVideos(channelId, options)` 또는 **공통 화면 컴포넌트**로 묶어서, 채널 ID·라벨(음악/영상)·외부 링크(YouTube Music / YouTube)만 props로 넘기기.  
  - 버그 수정·UX 변경 시 한 곳만 수정하면 되도록 유지보수성 향상.

### 1.5 썸네일 로드 실패

- **현상**: 썸네일 URL이 만료되거나 실패하면 빈/깨진 영역만 보일 수 있음.
- **제안**:  
  - `Image`에 `onError` 처리 후 **기본 placeholder 이미지** 또는 회색 박스 + “재생” 아이콘으로 대체.

### 1.6 API 할당량 초과 메시지

- **현상**: YouTube API `quotaExceeded` 시 사용자에게는 일반 “실패” 메시지로만 보임.
- **제안**:  
  - `services/youtube.ts`에서 에러 메시지에 `quotaExceeded` 또는 “daily limit” 포함 여부를 확인해, **“잠시 후 다시 시도해 주세요”** 같은 안내로 분기.

### 1.7 버전 번호 통일

- **현상**: `package.json`은 `1.0.0`, `app.config.js`는 `1.0.2`.
- **제안**:  
  - 한 곳(예: `app.config.js`)을 기준으로 두고, `package.json`의 `version`을 동일하게 맞추거나, 빌드 스크립트에서 app.config 버전을 참조하도록 정리.

### 1.8 접근성(플레이어)

- **현상**: 플레이어 화면에 “재생 중/일시정지” 등 상태 안내가 스크린 리더에 없을 수 있음.
- **제안**:  
  - 닫기 버튼은 이미 `accessibilityLabel` 있음.  
  - 재생/일시정지 상태를 **accessibilityLabel** 또는 **accessibilityState**로 전달해, 보조 기술 사용자가 상태를 알 수 있게 하기.

## 2. 기능 추천 (선택)

### 2.1 딥링크로 재생 화면 열기

- **현재**: `app.config.js`에 `scheme: 'cheezstereo'` 있음.
- **추천**:  
  - `cheezstereo://player/VIDEO_ID` 형태로 **딥링크** 지원.  
  - SNS·메신저에서 링크 클릭 시 앱이 설치되어 있으면 해당 영상 플레이어로 바로 이동.

### 2.2 공유하기

- **추천**:  
  - 음악/영상 카드 또는 플레이어 화면에서 **“공유”** 버튼 → `expo-sharing` 또는 `Share.share`로 `music.youtube.com/watch?v=...` / `youtube.com/watch?v=...` 링크 공유.  
  - 밴드 음악/영상 퍼뜨리기에는 효과적.

### 2.3 나중에 보기 / 즐겨찾기

- **추천**:  
  - `AsyncStorage` 등으로 **videoId 목록**만 로컬 저장.  
  - “나중에 보기” 탭 또는 설정 내 목록에서 바로 플레이어로 이동.  
  - 서버 없이 구현 가능.

### 2.4 다크 모드

- **현재**: `userInterfaceStyle: 'light'` 고정.
- **추천**:  
  - `userInterfaceStyle: 'automatic'`으로 두어 **시스템 설정 따라가기** 또는, 설정 화면에서 라이트/다크/시스템 선택.

### 2.5 채널 바로가기

- **추천**:  
  - 탭 헤더 우측 또는 “정보” 화면에 **YouTube 채널**, **YouTube Music 채널** 링크 버튼.  
  - `constants/youtube.ts`의 `YOUTUBE_URLS.musicChannel`, `videoChannel` 활용.

### 2.6 새 영상 알림 (선택)

- **추천**:  
  - 주기적으로 채널 최신 영상 확인 후 **로컬 알림**(expo-notifications) 또는 FCM으로 “새 곡/영상이 올라왔어요” 안내.  
  - 백엔드 없이 구현 시 할당량·배터리 고려 필요.
