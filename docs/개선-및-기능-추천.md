## 2. 남은 개선 항목

### 2.1 YouTube API 키 불필요 정리 (낮음)

- **현상**: `services/youtube.ts`와 `useChannelVideos.ts`가 잔존하지만 Music·Videos 탭에서 더 이상 사용하지 않음.
- **제안**: 해당 파일들을 Saved 탭에서도 쓰지 않는 것이 확인되면 제거 가능. 지금은 `getRelativeTime` 유틸 함수 때문에 유지.

### 2.2 package.json 버전 통일 (낮음)

- **현상**: `package.json`의 `version` 필드가 `app.config.js`의 `1.2.0`과 다를 수 있음.
- **제안**: `package.json`의 `version`을 `1.2.0`으로 동기화.

### 2.3 플레이어 재생 상태 접근성 (낮음)

- **현상**: 재생 중/일시정지 상태가 스크린 리더에 전달되지 않음.
- **제안**: `accessibilityState={{ busy: playing }}` 또는 `accessibilityLabel`을 재생 상태에 따라 변경.

## 3. 기능 추천 (미구현)

### 3.1 새 영상 알림 (높음)

- **추천**: 주기적으로 채널 최신 영상 확인 후 로컬 알림(expo-notifications) 또는 FCM으로 "새 영상이 올라왔어요" 안내.
- **난이도**: 상 — 백엔드 없이 구현 시 할당량·배터리 고려 필요.

### 3.2 재생 기록 (낮음)

- **추천**: 최근 재생한 트랙 목록을 AsyncStorage에 저장해 빠르게 다시 접근.
- **난이도**: 하 — PlayerContext에 history 배열 추가.

### 3.3 가사/곡 정보 (중간)

- **추천**: 트랙 상세 정보(작사·작곡·편곡)를 `constants/catalog.ts`의 Track 타입에 추가해 플레이어 화면 하단에 표시.
- **난이도**: 중 — 데이터 수집 필요.
