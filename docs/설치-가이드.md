# 앱 설치 가이드 및 "설치할 수 없음" 해결

사용자가 앱을 설치할 수 없다고 할 때 아래 항목을 순서대로 확인하세요.

---

## 1. Play Console 업로드 키 불일치 (가장 흔한 원인)

**증상**: 스토어에 AAB 업로드가 거부되거나, "이 기기에서는 앱을 설치할 수 없습니다" 메시지가 뜸.

**원인**: 빌드에 사용한 keystore의 서명과 Play Console에 등록된 업로드 인증서가 다름.

**해결**:

1. **현재 keystore 지문 확인**  
   `docs/android-signing.md`의 "keystore 지문 확인" 절대로 SHA1 확인.

2. **Play Console에 현재 키 등록**  
   - Play Console → 해당 앱 → **앱 무결성** 또는 **앱 서명**  
   - "업로드 키 재설정" 또는 "새 업로드 인증서 등록"  
   - `android-signing.md`의 "PEM 내보내기" 절대로 `upload_cert.pem` 생성 후 업로드.

3. **동일 keystore로 재빌드**  
   `eas build --platform android --profile production` 후 새 AAB만 업로드.

자세한 명령어와 값은 **`docs/android-signing.md`** 참고.

---

## 2. YouTube API "Android client application … are blocked"

**증상**: 앱 실행 시 `Requests from this Android client application <empty> are blocked` 오류.

**원인**: API 키에 "애플리케이션 제한사항"이 걸려 있어, 현재 앱(Expo Go 또는 개발 빌드)이 허용 목록에 없음.

**해결**:

1. [Google Cloud Console](https://console.cloud.google.com/) → **API 및 서비스** → **사용자 인증 정보**
2. 사용 중인 **API 키** 선택 → **애플리케이션 제한사항**
3. **제한 없음**으로 두면 Expo Go·개발·프로덕션 모두에서 동작 (키 유출만 주의).
4. 반드시 Android 앱으로 제한하려면:
   - **Expo Go**로 실행 시: 패키지명 `host.exp.exponent` + Expo Go SHA-1 추가
   - **본 앱 빌드** 사용 시: 패키지명 `com.cheez.projectcheezstereo` + 해당 keystore의 SHA-1 추가

## 3. APK 배포 후 "API 키가 설정되지 않았습니다"

**증상**: Expo Go에서는 정상인데, 배포한 APK를 설치해 실행하면 "YouTube API 키가 설정되지 않았습니다" 또는 "설정되어 있지 않습니다" 메시지가 뜸.

**원인**: APK/AAB는 EAS 서버에서 빌드됩니다. 로컬 `.env`는 서버에 없기 때문에, 빌드 시점에 `YOUTUBE_API_KEY`가 비어 있으면 앱에 키가 들어가지 않습니다.

**해결**:

1. **EAS Secret 추가**  
   터미널에서 (키는 실제 발급받은 값으로 교체):
   ```bash
   eas secret:create --name YOUTUBE_API_KEY --value "발급받은_YouTube_API_키" --scope project
   ```
   또는 [Expo Dashboard](https://expo.dev) → 해당 프로젝트 → **Secrets** → **Create** → 이름 `YOUTUBE_API_KEY`, 값에 API 키 입력.

2. **재빌드**  
   Secret만 추가해도 **이미 만든 APK에는 반영되지 않습니다.** 반드시 다시 빌드해야 합니다.
   ```bash
   eas build --platform android --profile production
   ```
   (preview 등 다른 프로필을 쓰면 해당 프로필로 빌드)

3. 새로 만든 APK/AAB를 배포하면 앱 실행 시 API 키가 정상 적용됩니다.

## 4. 로컬 빌드/설치 시 (개발자용)

**증상**: `npm run android` 또는 빌드된 APK 설치 시 실패.

**확인**:

| 항목 | 확인 방법 |
|------|-----------|
| **.env / API 키** | 프로젝트 루트에 `.env` 존재, `YOUTUBE_API_KEY=...` 설정 |
| **의존성** | `npm install` 또는 `npm install --legacy-peer-deps` 완료 |
| **Android 환경** | JDK 설치, `ANDROID_HOME` 설정, 기기/에뮬레이터 연결 |
| **캐시** | `npx expo start -c` 후 다시 실행 |

`.env`가 없다면 `.env.example`을 복사한 뒤 API 키를 입력하세요.

---

## 5. 스토어 배포 후 사용자 설치 불가

**확인**:

- **지원 Android 버전**: 앱이 요구하는 최소 SDK 버전보다 기기 OS가 낮으면 설치 불가. (Expo 기본값은 보통 API 24 이상.)
- **스토어 정책**: 앱이 해당 국가/기기에서 제한되지 않았는지 Play Console에서 확인.
- **저장 공간**: 기기에 여유 공간이 있는지 안내.

---

## 6. EAS 빌드 시 credentials 오류

**증상**: `eas build --platform android --profile production` 시 keystore를 찾을 수 없다는 오류.

**확인**:

- **eas.json**  
  `build.production.android.credentialsSource`가 `"local"`인지 확인.
- **credentials.json**  
  프로젝트 **루트**에 두고, `keystorePath`가 `credentials/android/keystore.jks`처럼 루트 기준 상대 경로인지 확인.
- **실제 파일**  
  `credentials/android/keystore.jks` 파일이 존재하는지 확인.

`credentials.json`은 비밀번호를 포함하므로 `.gitignore`에 포함되어 있어야 하며, 저장소에 커밋하지 마세요.

---

## 요약 체크리스트

- [ ] Play Console 업로드 키와 빌드 keystore 일치 (→ `android-signing.md`)
- [ ] `.env`에 `YOUTUBE_API_KEY` 설정 (로컬/Expo Go 실행 시)
- [ ] **APK/배포 빌드**: EAS Secrets에 `YOUTUBE_API_KEY` 추가 후 재빌드 (위 3번 참고)
- [ ] API 키 "Android … blocked" 시: Cloud Console에서 애플리케이션 제한사항 → 제한 없음 또는 해당 앱 추가 (위 2번 참고)
- [ ] `credentials.json` 위치 및 `keystore.jks` 존재 (EAS 로컬 서명 시)
- [ ] `npm install` 및 필요 시 `npx expo start -c` 완료

위를 모두 확인했는데도 설치가 안 되면, 발생한 **정확한 오류 메시지**와 **어느 단계에서** 실패하는지(로컬 설치 / 스토어 업로드 / 사용자 설치)를 기록해 두면 원인 파악에 도움이 됩니다.
