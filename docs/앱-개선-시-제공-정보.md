# 앱 개선 시 제공하면 좋은 정보 (상세)

앱 기능 확장·채널 변경·스토어 배포·브랜딩 변경 시, 아래 정보를 미리 준비해 두면 작업이 수월합니다.

---

## 1. 음악/영상 탭의 YouTube 채널 변경

### 1.1 채널 ID란?

- **채널 ID**: YouTube가 채널마다 부여한 고유 ID. 형식은 보통 `UC`로 시작하는 24자 문자열 (예: `UC8JozAPzI7Tbiivvv5icIFA`).
- 앱의 **음악 탭**은 이 ID로 YouTube Music 채널 영상 목록을 가져오고, **영상 탭**은 같은 ID(또는 다른 ID)로 YouTube 채널 영상 목록을 가져옵니다.

### 1.2 필요한 정보

| 구분 | 설명 | 예시 |
|------|------|------|
| **음악 탭용 채널 ID** | YouTube Music에서 보이는 채널의 ID | `UC8JozAPzI7Tbiivvv5icIFA` |
| **영상 탭용 채널 ID** | YouTube(일반)에서 보이는 채널의 ID. 음악과 같은 채널을 쓸 수도 있고, 별도 채널을 쓸 수도 있음 | `UC8JozAPzI7Tbiivvv5icIFA` 또는 다른 ID |

### 1.3 채널 ID 확인 방법

1. **YouTube 웹**에서 해당 채널 페이지로 이동 (예: `https://www.youtube.com/@채널핸들`).
2. 채널 페이지에서 **채널 소개** 또는 **영상** 등 아무 탭을 연 상태에서 **페이지 소스 보기** (우클릭 → "페이지 소스 보기") 또는 개발자 도구(F12)로 요소 검사.
3. 소스/요소에서 `"channelId":"UCxxxxxxxxxxxxxxxxxxxxxx"` 형태로 검색하면 채널 ID를 찾을 수 있음.
4. 또는 [YouTube Data API v3](https://developers.google.com/youtube/v3/docs)를 사용해 채널 핸들(`@username`)로 채널 정보를 조회하면 `id` 필드에 채널 ID가 나옵니다.

### 1.4 코드에서 쓰이는 위치

- **파일**: `constants/youtube.ts`
- **변수**: `YOUTUBE_CHANNELS.music`, `YOUTUBE_CHANNELS.videos`
- **사용처**:
  - `app/(tabs)/music.tsx` → `YOUTUBE_CHANNELS.music` (음악 탭 목록)
  - `app/(tabs)/videos.tsx` → `YOUTUBE_CHANNELS.videos` (영상 탭 목록)

변경 시 `YOUTUBE_CHANNELS`의 `music`, `videos` 값만 바꿔주면 됩니다.

## 2. 채널 링크(URL/핸들) — 헤더 메뉴용

### 2.1 무엇에 쓰이나?

- 앱 **헤더 메뉴**(점 세 개)에서 "YouTube 채널", "YouTube Music 채널"을 누르면 **어느 URL로 브라우저가 열리는지**를 결정합니다.
- **파일**: `constants/youtube.ts`
- **변수**: `YOUTUBE_URLS.musicChannel`, `YOUTUBE_URLS.videoChannel`

### 2.2 필요한 정보

| 구분 | 설명 | 형식 | 현재 값(예시) |
|------|------|------|----------------|
| **YouTube Music 채널 URL** | 음악 채널로 이동할 링크 | `https://music.youtube.com/channel/채널ID` | `musicChannel`은 위 `YOUTUBE_CHANNELS.music`으로 자동 생성됨 |
| **YouTube(일반) 채널 URL** | 일반 YouTube 채널로 이동할 링크 | `https://www.youtube.com/@핸들` 또는 `https://www.youtube.com/channel/채널ID` | `https://www.youtube.com/@cheezstereo` |

### 2.3 제공하면 좋은 값

- **YouTube(일반) 채널**  
  - **핸들**만 알려주면 됨: 예) `@cheezstereo` → 코드에선 `https://www.youtube.com/@cheezstereo` 형태로 사용.  
  - 또는 **채널 ID**를 쓰고 싶다면: `https://www.youtube.com/channel/UCxxxxxxxxxx` 형태로 전체 URL을 알려주면 됨.
- **YouTube Music 채널**  
  - 보통은 **채널 ID**만 있으면 됨. 현재는 `YOUTUBE_CHANNELS.music`으로 `https://music.youtube.com/channel/채널ID`를 자동 생성하므로, 채널 ID만 맞추면 됨.

### 2.4 사용되는 화면

- `app/(tabs)/_layout.tsx`  
  - "YouTube 채널" → `YOUTUBE_URLS.videoChannel`  
  - "YouTube Music 채널" → `YOUTUBE_URLS.musicChannel`

## 3. 스토어 배포·서명 (Play Console 업로드)

### 3.1 언제 필요하나?

- **APK/AAB를 Google Play에 처음 올릴 때**, 또는 **"이 기기에서는 앱을 설치할 수 없습니다"** / 업로드 키 불일치 오류가 날 때 필요합니다.
- EAS Build를 쓰면 EAS가 keystore를 관리할 수도 있고, **로컬 keystore**를 쓰도록 설정할 수도 있습니다.

### 3.2 필요한 정보 (로컬 keystore 사용 시)

| 항목 | 설명 | 예시 |
|------|------|------|
| **keystore 파일 경로** | `.jks` 또는 `.keystore` 파일 위치 | `credentials/android/keystore.jks` |
| **keystore 비밀번호** | keystore 열 때 쓰는 비밀번호 | (본인만 아는 값) |
| **key alias** | keystore 안에 있는 키의 별칭 | `upload` 등 |
| **key 비밀번호** | 해당 alias 키의 비밀번호 (keystore와 같을 수 있음) | (본인만 아는 값) |

### 3.3 Play Console에 등록할 때 필요한 값

- **업로드 키 인증서**  
  - keystore에서 **SHA-1** (및 필요 시 PEM)을 뽑아서 Play Console **앱 서명** / **앱 무결성**에서 "업로드 키"로 등록합니다.
- **SHA-1 확인 예시 (로컬)**  
  - `keytool -list -v -keystore credentials/android/keystore.jks -alias upload`  
  - 출력에서 "SHA1:" 뒤의 지문을 복사해 Play Console에 등록.

### 3.4 EAS만 쓸 때

- EAS가 keystore를 관리하면, **EAS Dashboard** 또는 `eas credentials`에서 확인 가능.
- Play Console에는 **EAS가 사용하는 업로드 키의 인증서(SHA-1/PEM)**를 등록해야 합니다. 자세한 절차는 Expo 문서의 [Android credentials](https://docs.expo.dev/app-signing/managed-credentials/) 및 프로젝트의 `docs/설치-가이드.md`를 참고하세요.

### 3.5 요약

- **제공할 수 있는 것**:  
  - "로컬 keystore 쓴다" → keystore 경로, alias, 비밀번호(보안 유의).  
  - "EAS 관리한다" → EAS에서 내보낸/표시되는 업로드 인증서 정보.  
- **Play Console 측**: 항상 **현재 빌드에 사용한 업로드 키의 SHA-1(또는 PEM)**이 등록되어 있어야 스토어 업로드·설치가 됩니다.

## 4. 앱 이름·버전·패키지 (브랜딩/버전 관리)

### 4.1 앱 이름 (사용자에게 보이는 이름)

- **위치**: `app.config.js` → `name`
- **현재 값**: `'치즈스테레오'`
- **변경 시**: 앱 설치 후 기기에서 보이는 이름이 바뀝니다. 원하는 표기(예: "치즈스테레오", "CheezStereo")를 알려주면 됩니다.

### 4.2 버전 번호

- **위치**: `app.config.js` → `version`
- **현재 값**: `'1.1.1'`
- **형식**: 보통 `주요.부요.패치` (예: `1.2.0`). 스토어는 새 APK/AAB 업로드 시 이전보다 큰 버전이어야 할 수 있음.
- **제공할 것**: "다음 버전을 1.2.0으로 올린다"처럼 원하는 버전 문자열.

### 4.3 패키지명 / 번들 ID (Android / iOS)

- **Android**: `app.config.js` → `android.package`  
  - 현재: `com.cheez.projectcheezstereo`
- **iOS**: `app.config.js` → `ios.bundleIdentifier`  
  - 현재: `com.cheez.projectcheezstereo`
- **주의**: 한 번 스토어에 올린 앱의 패키지명/번들 ID를 바꾸면 **새 앱**으로 취급되어 기존 앱과 업데이트가 이어지지 않습니다. 변경 시 "새 앱으로 출시한다"는 전제로 진행해야 합니다.
- **제공할 것**: 변경한다면 원하는 패키지명/번들 ID (예: `com.밴드명.앱이름`).

### 4.4 앱 설명 (스토어용)

- **위치**: `app.config.js` → `description`
- **현재 값**: `'치즈스테레오 밴드의 음악과 영상을 한곳에서. YouTube 채널 연동.'`
- **제공할 것**: 스토어에 넣을 짧은 설명 문구.

### 4.5 딥링크 scheme

- **위치**: `app.config.js` → `scheme`
- **현재 값**: `'cheezstereo'`  
  - 예: `cheezstereo://player/VIDEO_ID` 형태로 앱 열기
- **변경 시**: 외부에서 앱을 여는 링크가 바뀝니다. 이미 공개했다면 변경 시 주의 필요. 새로 정할 scheme이 있으면 알려주면 됩니다.

## 5. 정리 체크리스트

앱 개선·채널 변경·스토어 작업 시 아래를 미리 정리해 두면 좋습니다.

| 목적 | 제공하면 좋은 정보 |
|------|---------------------|
| **음악/영상 탭 채널 변경** | 음악용 채널 ID, 영상용 채널 ID (각각 문자열). |
| **헤더 채널 링크 변경** | YouTube(일반) 채널 URL 또는 핸들(`@이름`). YouTube Music은 채널 ID만 맞추면 됨. |
| **스토어 업로드/서명** | 로컬 keystore 사용 시: 경로, alias, 비밀번호. EAS 사용 시: EAS 업로드 키 정보. Play Console에는 해당 업로드 키의 SHA-1/PEM 등록. |
| **앱 이름 변경** | 표시 이름 (예: 치즈스테레오). |
| **버전 올리기** | 원하는 버전 번호 (예: 1.2.0). |
| **패키지명/번들 ID 변경** | 원하는 Android 패키지명, iOS 번들 ID (새 앱으로 출시할 때만). |
| **스토어 설명** | 앱 설명 한 줄. |
| **딥링크 scheme** | 사용할 scheme (예: cheezstereo). |

이 문서는 `docs/앱-개선-시-제공-정보.md`로 저장되어 있으며, 필요 시 팀과 공유하거나 작업 시 참고하면 됩니다.
